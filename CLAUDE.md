## Commands

```bash
# Build
./gradlew build

# Run all tests
./gradlew test

# Run a single test class
./gradlew test --tests "com.example.ToyProject_Board.domain.post.PostServiceTest"

# Run a single test method
./gradlew test --tests "com.example.ToyProject_Board.domain.post.PostServiceTest.createSuccess"

# Start MySQL via Docker (required before running the app)
docker-compose up -d

# Run the application
./gradlew bootRun
```

The app requires `DB_URL`, `DB_USERNAME`, `DB_PASSWORD`, and `DB_NAME` environment variables set before running.

## Architecture

**Stack:** Spring Boot 4.x, Java 17, Spring Security, Spring Data JPA, MySQL, JWT (jjwt 0.12.3), Lombok

**Package layout:** `com.example.ToyProject_Board`
- `domain/` — feature packages (`user/`, `post/`), each containing `controller/`, `service/`, `repository/`, `dto/`, and the entity class
- `global/` — cross-cutting concerns: `jwt/` (filter + util), `config/`, `security/`

**Auth flow:**
- `POST /api/auth/**` is public; `GET /api/posts/**` is public; everything else requires a JWT
- `JwtAuthenticationFilter` extracts the Bearer token, validates it, and sets `userId` (Long) as both a `SecurityContext` principal and a `request` attribute
- `UserService.login()` issues access + refresh tokens; `reissue()` implements RTR (Refresh Token Rotation) by invalidating the old refresh token on use; refresh token is stored on the `User` entity

**Testing conventions:**
- Service tests use `@ExtendWith(MockitoExtension.class)` with `@InjectMocks` + `@Mock` — pure unit tests, no Spring context
- Controller tests use `@WebMvcTest` + `@Import(SecurityConfig.class)` and extend `ControllerTestSupport`
- `ControllerTestSupport` mocks `JwtAuthenticationFilter` via `@MockitoBean` and pre-sets authentication as `userId = 1L`; call `setAuthentication(userId)` inside a test to override the principal
- `ReflectionTestUtils.setField()` is used to set private `id` fields on entities (no public setter, id is generated by JPA)
